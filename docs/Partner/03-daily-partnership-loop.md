# 03 - Daily Partnership Loop

Navigation: Previous `02-first-hour.md` | Index `README.md` | Next `04-how-to-ask-agent1.md`

The best Agent1 workflow feels like a rhythm, not a fire drill.

## Morning

- Review active jobs and anomalies.
- Set priorities for the day.
- Confirm runtime mode and scope (active/shadow/dry_run).

## Midday

- Let Agent1 run execution loops.
- Intervene only on decision points, not routine mechanics.
- Resolve clarification requests quickly to keep flow moving.

## Afternoon

- Review PR and issue outcomes.
- Capture unresolved work as explicit next actions.
- Decide what graduates from sandbox confidence to production confidence.

## End Of Day

- Leave intent, not just status.
- A good handoff says: what matters, what changed, what is risky, what is next.

Agent1 carries that forward without losing state continuity.

## The Partnership KPI

If you can return after context-switching and recover in minutes, the loop is healthy.
