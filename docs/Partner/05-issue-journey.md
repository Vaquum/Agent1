# 05 - Issue Journey

Navigation: Previous `04-how-to-ask-agent1.md` | Index `README.md` | Next `06-pr-journey.md`

Issue work is where partnership trust is built fastest.

## Typical Flow

1. Trigger: mention or assignment.
2. Context check: execute or request clarification.
3. Execution: deterministic state advancement.
4. Feedback loop: continue until resolved or blocked with reason.

## What Good Looks Like

- No hidden fallbacks.
- Explicit transitions and reasons.
- Clear human handoff points.
- Durable history you can replay.

## Clarification Loop

If context is insufficient, Agent1 should block safely and ask.

When you provide new context, Agent1 resumes from the blocked state instead of restarting from zero.

## Terminal Outcomes

- Completed with expected result.
- Blocked with explicit cause and next requirement.
- Escalated by human decision.
