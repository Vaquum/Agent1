# Agent1 Partner Guide

You are not "using a tool." You are partnering with a software engineer that never sleeps.

This guide is a short linked sequence. Each page is intentionally brief. Read in order the first time, then jump where you need.

## Reading Order

1. `01-your-role.md` - what your role is, and what Agent1's role is.
2. `02-first-hour.md` - the first real collaboration loop.
3. `03-daily-partnership-loop.md` - how to run Agent1 day after day.
4. `04-how-to-ask-agent1.md` - prompts and requests that produce great outcomes.
5. `05-issue-journey.md` - how issue work flows from mention to done.
6. `06-pr-journey.md` - how PR work flows until human terminal decision.
7. `07-dashboard-signals.md` - how to read the system in real time.
8. `08-safety-model.md` - the boundaries that keep trust intact.
9. `09-sandbox-to-production.md` - how to move from safe sandbox to live confidence.
10. `10-incident-rhythm.md` - what to do when the system signals risk.
11. `11-future-horizons.md` - where this partnership goes next.

- Human decisions stay human.
- Agent1 carries workload, continuity, and execution pressure.
- Transparency is default.
- Safety is policy-backed, not wish-based.
