# 04 - How To Ask Agent1

Navigation: Previous `03-daily-partnership-loop.md` | Index `README.md` | Next `05-issue-journey.md`

Great outputs start with sharp intent.

## Ask In Four Parts

1. **Goal** - what outcome you want.
2. **Scope** - where Agent1 should operate.
3. **Constraints** - what it must not do.
4. **Done signal** - how you will judge completion.

## Prompt Shapes That Work

- "Do X in this issue, stay inside sandbox scope, report decisions."
- "Prioritize reliability over speed, show your state transitions."
- "If context is missing, ask me before acting."

## Prompt Shapes To Avoid

- Vague urgency without scope.
- Implicit permission for risky actions.
- Hidden acceptance criteria.

## Clarification Is A Feature

When Agent1 asks follow-up questions, that is safety working as designed.

You are seeing policy and determinism prevent premature execution.
